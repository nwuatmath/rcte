Overview: User Path for RCE Using Matching

Objective: This set of web pages show the path a user would take in order to complete a backward-looking matched comparison design. These pages are the web versions of prototypes reviewed at the June TWG meeting. The first few steps, including discovery and account creating, are intentionally omitted for now.

The tools will be connected. For example, in step 2 the user will tell us the name of the technology that is being evaluated. This answer will be recorded on the database and we will use it to personalize other tools. Similarly, based on the answers to step 1 we will show the user different research paths. The wizard architecture is in progress and will provide the structure to link the pieces together.
